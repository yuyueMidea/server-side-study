Jenkins:用来实现 “开发写完代码 → 构建 → 测试 → 打包 → 部署到服务器”， 整个流程自动化。

Jenkins发版整体流程：开发 → 推送代码 → Jenkins 自动触发构建 → 编译打包 → 部署到服务器

典型阶段（Pipeline）
- 拉取代码：从 GitHub/GitLab 拉最新代码；
- 构建：执行 npm build / go build / mvn package 等；
- 测试（可选）：单元测试、lint 检查；
- 打包与归档：打包输出物；
- 部署：上传到目标服务器（或触发 Docker 构建 + 容器启动）；
- 通知：构建结果通知（钉钉 / 企业微信 / 邮件）。
